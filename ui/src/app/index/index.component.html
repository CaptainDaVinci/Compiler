About

Compiler flag selection can be an effective way to increase the quality of executable code according to different code quality criteria. Evolutionary algorithms have been successfully applied to this optimization problem. In many situations, executable code can be significantly optimized at the time of compilation. Code optimization can produce significant gains in terms of the time needed to solve practical problems and other resources (e.g. energy, memory, etc) required to solve them.


In GCC, different optimization levels group combination of several flags are used to define different trade-offs between the criteria that serve to describe the quality of the code. The different levels of optimization options available in GCC are,


  • O0: No optimization.
  • O1: Basic optimization level. 39 (28) variables.
  • O2: Recommended optimization level. Includes all flags existing in -O1. 73 (54) variables.
  • O3: Highest level of optimization. 82 (58) variables.
